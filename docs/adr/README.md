# ADR

У цій директорії зберігаються записи архітектурних рішень для цього проєкту.

## Формат запису

ADR варто записувати у хронологічному порядку. Кожен новий ADR створюється як окремий файл з префіксом-номером.

**Назва файлу:** `<порядковий номер>-<об'єкт вибору>.md`

**Структура файлу:**

```text
# ADR-<порядковий номер>: <об'єкт вибору> — <варіанти>

**Статус:** <статус ухвалення>
**Дата:** <дата ADR>
**Автор:** <автор(и) ADR>

## Контекст

## Рішення

**<варіант 1> — < відхилено | обрано >**
[Коротке обґрунтування]

## Наслідки

## Подальші міркування

```

## Взаємодія з проєктом

- Відкрийте PR, додавши новий файл ADR у дану директорію.
- Початковий статус — `Запропоновано`.
- Після рев'ю та злиття змініть статус на `Ухвалено`.
- Щоб переглянути рішення: напишіть новий ADR із вищим номером, встановіть його статус як `Ухвалено`, а статус старого ADR позначте як `Замінено на ADR-NNNN`.
- Ніколи не змінюйте обґрунтування старого ADR.
