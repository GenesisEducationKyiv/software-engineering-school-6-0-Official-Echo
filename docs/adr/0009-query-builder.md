# ADR-0009: Query builder — Kysely vs інші

**Статус:** Ухвалено\
**Дата:** 12.04.2026\
**Автор:** Official Echo

## Контекст

Після вибору SQLite та `better-sqlite3` потрібен був шар для побудови SQL-запитів без прямої взаємодії із SQL. Рядкові SQL-запити були відхилені як такі, що не масштабуються і схильні до помилок. Потрібна була безпека типів без ваги повноцінного ORM.

## Рішення

**Prisma — відхилено**

Prisma — повноцінний ORM з власним мовою схеми, окремим процесом генерації клієнта та власним движком запитів. Ця вага невиправдана для проєкту з однією таблицею. Підтримка SQLite в Prisma також обмежена порівняно з PostgreSQL, і Prisma не надає прямого доступу до синхронного API `better-sqlite3`.

**Knex — відхилено**

Knex є зрілим і широко використовуваним, але не має вбудованої безпеки — типи є стороннім доповненням, що неодноразово відставали від реального API. Без TypeScript це менш актуально, але Knex також важчий за Kysely і менш активно розвивається.

**Drizzle — відхилено**

Drizzle є сучасним конкурентом Kysely з хорошою підтримкою SQLite та схожим підходом до типобезпеки. Однак на момент прийняття рішення Kysely мав більш зрілий API і ширшу екосистему для `better-sqlite3` зокрема.

**Kysely — обрано**

Kysely є де-факто стандартом типобезпечних query builder'ів для Node.js без ORM. Його API є декларативним і зчитується майже як SQL: `.selectFrom().select().where().execute()`. SQLite підтримується офіційно. Для проєкту без TypeScript Kysely все одно забезпечує чистий, компонований API запитів.

## Наслідки

- Усі запити до бази даних живуть у `src/repositories/` і будуються через Kysely API
- Схема таблиці описана в `src/db/kysely.js` як тип інтерфейсу для документування структури
- Міграції виконуються через Kysely schema builder при запуску, без зовнішнього міграційного CLI

## Подальші міркування

Якщо проєкт перейде на TypeScript, Kysely розкриє свою повну цінність — повна типобезпека на рівні запитів без кодогенерації. Drizzle варто переглянути при такій міграції, оскільки його підтримка TypeScript також є першокласною.
