# ADR-0002: HTTP-фреймворк — Express vs Fastify

**Статус:** Ухвалено\
**Дата:** 12.04.2026\
**Автор:** Official Echo

## Контекст

Сервіс надає чотири REST-ендпоінти плюс маршрути для здоров'я та метрик. Завдання явно забороняло фреймворки високого рівня (NestJS було названо окремо) та явно дозволяло Express і Fastify як два допустимі варіанти для Node.js. Вибір був бінарним.

## Рішення

**Fastify — відхилено**

Fastify — фреймворк, з яким розробник не мав попереднього досвіду. На відміну від рішення щодо SQLite, де попередня знайомість спрощувала вибір, архітектура плагінів Fastify, система валідації на основі схем та відмінна модель життєвого циклу представляли суттєву незнайому територію. Впровадження його в проєкт з обмеженим часом сповільнило б розробку та збільшило ризик тонкого неправильного використання.

**Express — обрано**

Express — найдокументованіший HTTP-фреймворк для Node.js. Його модель middleware, `Router` та сигнатура обробника помилок добре зрозумілі та читабельні для будь-якого Node.js-розробника. Проєкт використовує Express 5, який нативно підтримує поширення асинхронних помилок. Необроблені відхилення `Promise`\`ів у обробниках маршрутів автоматично передаються до обробника помилок, усуваючи потребу в `try/catch` у кожному асинхронному маршруті. Стек middleware природно відповідає лінійному ланцюжку Express.

## Наслідки

- Нативна обробка асинхронних помилок в Express 5 тримає обробники маршрутів чистими
- Патерн middleware є явним і читабельним: кожна відповідальність — окрема функція в ланцюжку
- Екосистема Express має безліч плагінів та патернів `middleware`, у проєкті використано `prom-client`
- Перевага Fastify в пропускній здатності не має значення в цьому масштабі

## Подальші міркування

Якби сервіс став критичним з точки зору продуктивності або якби валідація запитів/відповідей на основі схем стала вимогою, Fastify варто було б переглянути. Його інтеграція з OpenAPI та валідація JSON Schema є реальними перевагами в API-орієнтованих сервісах. Для поточного обсягу Express залишається відповідним вибором.
